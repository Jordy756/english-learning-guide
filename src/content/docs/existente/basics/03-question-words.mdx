Cuando las preguntas básicas de "sí o no" no son suficientes para obtener lo que buscas, entran en juego las **Question Words**. Estas palabras nos permiten pedir detalles específicos como nombres, lugares, motivos, momentos exactos o la manera en que se hace algo.

### Tu diccionario de Question Words

Aquí tienes las palabras interrogativas más comunes. Piénsalas como herramientas que te ayudan a extraer la información exacta que necesitas de una conversación.

<Tabs>
  <TabItem label="What">
```javascript
const what = {
  meaning: "Qué / Cuál",
  definition: "Para pedir información sobre cosas o ideas.",
  examples: [
    { en: "What is your favorite color?", es: "¿Cuál es tu color favorito?" },
    { en: "What do you want to eat?", es: "¿Qué quieres comer?" },
    { en: "What does she do for a living?", es: "¿A qué se dedica ella?" },
    { en: "What happened yesterday?", es: "¿Qué pasó ayer?" },
    { en: "What makes you happy?", es: "¿Qué te hace feliz?" },
    { en: "What does your brother look like?", es: "¿Cómo es físicamente tu hermano?" }
  ]
};
```
    <Aside type="tip" title="¿Cuándo usar What o Which?">
      Usa **What** cuando las opciones son abiertas o infinitas (como preguntar por un color favorito). Por el contrario, verás que usamos **Which** cuando hay un grupo limitado de opciones para elegir.
    </Aside>

### Las diferentes combinaciones de "How"

La palabra `How` es sumamente versátil. Por sí sola significa "cómo", pero tiene el poder de unirse a adjetivos o adverbios para crear preguntas totalmente nuevas sobre cantidad, frecuencia o duración.

<Tabs>
  <TabItem label="How">
```javascript
const how = {
  meaning: "Cómo",
  definition: "Para preguntar sobre la manera o el modo en que se hace algo.",
  examples: [
    { en: "How is your mother?", es: "¿Cómo está tu madre?" },
    { en: "How do you fix this?", es: "¿Cómo arreglas esto?" },
    { en: "How does she feel today?", es: "¿Cómo se siente ella hoy?" },
    { en: "How about a cup of coffee?", es: "¿Qué tal una taza de café?" },
    { en: "How far is the airport from here?", es: "¿Qué tan lejos está el aeropuerto de aquí?" },
    { en: "How come you are so late?", es: "¿Cómo es que vienes tan tarde? (Coloquial)" }
  ]
};
```
  </TabItem>

<Aside type="tip" title="How much vs How many">
  La diferencia es simple:
  - **How much**: Para cosas que no puedes contar individualmente (líquidos, dinero, tiempo)
  - **How many**: Para cosas que puedes contar una por una (personas, objetos, días)

  Pregúntate: ¿Puedo decir "uno, dos, tres..." con esto? Si sí utiliza **how many**. Si no utiliza **how much**.
</Aside>

### Cómo construir tus preguntas

Para que tu pregunta se entienda correctamente, necesitas seguir un orden lógico. Es como seguir una receta: si cambias el orden de los ingredientes, el resultado no será el mismo.

La estructura estándar que debes seguir es: **Question Word + Auxiliar + Sujeto + Verbo principal + Complemento + ?**

<Aside title="Una regla especial con el verbo 'to be'">
  Como ya aprendiste en lecciones anteriores, el verbo **to be** (am, is, are, was, were) es independiente y no necesita la ayuda de auxiliares como `do` o `does`. En estos casos, la Question Word va seguida directamente por el verbo to be.

  **Ejemplos correctos:**
  - "Where **are** you from?" ✓
  - "Who **is** she?" ✓

  **Evita este error:**
  - ~~"Where **do** you **are** from?"~~ ✗
</Aside>

### Situaciones especiales

A veces, las reglas cambian un poco dependiendo de lo que estemos preguntando. Aquí tienes dos casos muy comunes que te harán sonar mucho más natural:

<Card title="Preguntas sobre el sujeto" icon="document">
  Si la Question Word es la que realiza la acción, **no necesitas usar auxiliares** (do/does). 
  
```javascript
// Pregunta sobre el sujeto (Who hace la acción)
const question = "Who is calling?"; // Quien está llamando? (No necesitas "do" porque "Who" es el sujeto)
```
</Card>
<Card title="Preposiciones al final" icon="bars">
  En inglés es muy común (y natural) dejar las preposiciones al final de la frase.

```javascript
// Pregunta con preposición al final
const question = "Who are you talking to?"; // ¿Con quién estás hablando?
```
</Card>

## Errores habituales que debes evitar

Es normal cometer errores al principio. Aquí tienes una lista de "bugs" comunes que podrías encontrar al construir tus preguntas y cómo solucionarlos.

<Aside type="danger" title="Error de posición">
  La **Question Word** SIEMPRE debe ir al puro inicio de tu oración. Si la pones al final, la pregunta perderá su estructura formal.

  ```javascript
  // ❌ Incorrecto
  console.log("You live where?");         // Estilo muy informal que debes evitar

  // ✅ Correcto
  console.log("Where do you live?");      // La palabra interrogativa abre la pregunta
  ```
</Aside>

<Aside type="danger" title="Error con el verbo 'to be'">
  Uno de los errores más frecuentes es intentar usar `do` o `does` con el verbo to be. Recuerda que ellos no se llevan bien en la misma oración.

  ```javascript
  // ❌ Incorrecto
  console.log("Who does she is?");           

  // ✅ Correcto
  console.log("Who is she?");                
  ```
</Aside>

<Aside type="danger" title="Confusión entre Whose y Who's">
  Aunque se pronuncian exactamente igual, tienen funciones muy distintas. Asegúrate de elegir la correcta según lo que quieras decir.

  ```javascript
  // ❌ Incorrecto: Usar who's para posesión
  console.log("Who's book is this?");        // Estás preguntando "¿Quién es libro es este?"

  // ✅ Correcto: Usar whose para decir "de quién"
  console.log("Whose book is this?");        
  ```
</Aside>

<Aside type="danger" title="Uso incorrecto de How much/many">
  Elegir el tipo de cantidad equivocado puede confundir a quien te escucha. Fíjate bien si lo que preguntas se puede contar por unidades o no.

  ```javascript
  // ❌ Incorrecto
  console.log("How many water do you need?");  // El agua no se cuenta por unidades
  console.log("How much apples do you want?"); // Las manzanas sí se pueden contar

  // ✅ Correcto
  console.log("How much water do you need?");  
  console.log("How many apples do you want?"); 
  ```
</Aside>
