Los números en inglés son como tipos primitivos en JavaScript: esenciales y fundamentales para comunicarte en cualquier contexto. Desde dar tu edad y número de teléfono hasta hablar de precios, fechas y cantidades, dominar los números te permite expresar información cuantitativa con precisión.

## Números del 0 al 20

Estos son los números base que debes memorizar completamente, ya que forman la base de todos los demás:

```js
const basicNumbers = {
  0: "zero",      1: "one",       2: "two",
  3: "three",     4: "four",      5: "five",
  6: "six",       7: "seven",     8: "eight",
  9: "nine",      10: "ten",      11: "eleven",
  12: "twelve",   13: "thirteen", 14: "fourteen",
  15: "fifteen",  16: "sixteen",  17: "seventeen",
  18: "eighteen", 19: "nineteen", 20: "twenty"
};
```

<Aside type="tip" title="Patrón 13-19">
Del 13 al 19, el patrón general es: **número base + "teen"**
- Three → Thirteen
- Four → Fourteen
- Five → Fifteen (cambia a "fif")
- Six → Sixteen

Excepciones: eleven, twelve (no siguen ningún patrón)
</Aside>

## Decenas y Números Compuestos

Las decenas siguen un patrón consistente, y para números compuestos (21-99) usas guion:

<Tabs>
  <TabItem label="Decenas (10-90)">

```js
const tens = {
  10: "ten",
  20: "twenty",
  30: "thirty",    // three → thirty
  40: "forty",     // four → forty (NO "fourty")
  50: "fifty",     // five → fifty
  60: "sixty",
  70: "seventy",
  80: "eighty",
  90: "ninety"
};
```

**Patrón:** Número base + "ty" (con algunas variaciones ortográficas)

  </TabItem>

  <TabItem label="Compuestos (21-99)">

Para números entre las decenas, usa guion:

```js
const compositeNumber = (tens, ones) => {
  return `${tens}-${ones}`;
};

// Ejemplos
console.log(compositeNumber("twenty", "one"));   // "twenty-one"
console.log(compositeNumber("thirty", "five"));  // "thirty-five"
console.log(compositeNumber("ninety", "nine"));  // "ninety-nine"
```

**Ejemplos completos:**
  - 21: twenty-one _(veintiuno)_
  - 47: forty-seven _(cuarenta y siete)_
  - 63: sixty-three _(sesenta y tres)_
  - 89: eighty-nine _(ochenta y nueve)_

  </TabItem>
</Tabs>

<Aside type="caution" title="El número 40 se escribe sin u">
El número 40 se escribe **"forty"** (sin "u"). Es la única decena que pierde la "u" del número base.

- Fourty ⛔
- **Forty** ✅
</Aside>

## Números Grandes

Para números grandes, se usan "hundred", "thousand", "million" sin plural cuando van precedidos de un número:

```js
const largeNumbers = {
  100: "one hundred",         // cien
  200: "two hundred",         // doscientos (NO "two hundreds")
  1000: "one thousand",       // mil
  5000: "five thousand",      // cinco mil (NO "five thousands")
  1000000: "one million",     // un millón
  10000000: "ten million"     // diez millones
};

// Para números compuestos, usa "and" después de hundred
const buildNumber = (hundreds, tens) => {
  return `${hundreds} hundred and ${tens}`;
};

console.log(buildNumber("three", "forty-five")); 
// "three hundred and forty-five" (345) - trescientos cuarenta y cinco
```

**Estructura de números grandes:**
- **456:** four hundred and fifty-six _(cuatrocientos cincuenta y seis)_
- **1,234:** one thousand, two hundred and thirty-four _(mil doscientos treinta y cuatro)_
- **50,000:** fifty thousand _(cincuenta mil)_
- **1,000,000:** one million _(un millón)_

<Aside type="note" title="Diferencia UK vs US">
En inglés británico, se usa "and" después de "hundred":  
- UK: "two hundred **and** fifty"
- US: "two hundred fifty" (sin "and")

Ambas formas son correctas, pero la versión británica es más común globalmente.
</Aside>

## Números Ordinales

Los ordinales indican posición u orden. La mayoría añaden "th", pero los tres primeros son irregulares:

<Tabs>
  <TabItem label="1st a 10th">

```js
const ordinals = {
  1: "first",     // 1st - irregular
  2: "second",    // 2nd - irregular
  3: "third",     // 3rd - irregular
  4: "fourth",    // 4th
  5: "fifth",     // 5th
  6: "sixth",     // 6th
  7: "seventh",   // 7th
  8: "eighth",    // 8th
  9: "ninth",     // 9th
  10: "tenth"     // 10th
};
```

  </TabItem>

  <TabItem label="Patrón General">

Para formar ordinales, el patrón es: número cardinal + "th"

```js
const makeOrdinal = (cardinal) => {
  const exceptions = {
    "one": "first",
    "two": "second",
    "three": "third",
    "five": "fifth",    // cambia a "fifth"
    "twelve": "twelfth" // cambia a "twelfth"
  };
  
  if (exceptions[cardinal]) {
    return exceptions[cardinal];
  }
  
  return cardinal + "th"; // patrón general
};

// Ejemplos
console.log(makeOrdinal("four"));     // "fourth"
console.log(makeOrdinal("thirteen")); // "thirteenth"
console.log(makeOrdinal("twenty"));   // "twentieth"
```

**Ordinales compuestos:**
  - 21st: twenty-first _(vigésimo primero)_
  - 32nd: thirty-second _(trigésimo segundo)_
  - 43rd: forty-third _(cuadragésimo tercero)_
  - 54th: fifty-fourth _(quincuagésimo cuarto)_

  </TabItem>
</Tabs>

## Operaciones Matemáticas

```js
const mathOperations = {
  addition: "5 + 3 = 8",       // five plus three equals eight (cinco más tres es igual a ocho)
  subtraction: "10 - 4 = 6",   // ten minus four equals six (diez menos cuatro es igual a seis)
  multiplication: "3 × 2 = 6", // three times two equals six (tres por dos es igual a seis)
  division: "8 ÷ 2 = 4"        // eight divided by two equals four (ocho dividido por dos es igual a cuatro)
};
```

**Vocabulario matemático:**
- **+** : plus _(más)_
- **-** : minus _(menos)_
- **×** : times / multiplied by _(por / multiplicado por)_
- **÷** : divided by _(dividido por)_
- **=** : equals / is _(es igual a / es)_

## Fracciones

Las fracciones usan ordinales para el denominador:

<Tabs>
  <TabItem label="Fracciones Comunes">

```js
const commonFractions = {
  "1/2": "one half / a half",         // un medio (excepción)
  "1/4": "one quarter / a quarter",   // un cuarto (excepción)
  "3/4": "three quarters",            // tres cuartos
  "1/3": "one third",                 // un tercio
  "2/3": "two thirds",                // dos tercios
  "1/5": "one fifth",                 // un quinto
  "2/5": "two fifths"                 // dos quintos
};
```

  </TabItem>

  <TabItem label="Patrón General">

**Estructura:** numerador (cardinal) + denominador (ordinal plural si > 1)

```js
const makeFraction = (numerator, denominator) => {
  const ordinal = makeOrdinal(denominator);
  const plural = numerator > 1 ? ordinal + "s" : ordinal;
  return `${numerator} ${plural}`;
};

// Ejemplos
console.log(makeFraction(1, "eighth")); // "1 eighth"
console.log(makeFraction(3, "eighth")); // "3 eighths"
console.log(makeFraction(5, "sixth"));  // "5 sixths"
```

  </TabItem>
</Tabs>

## Errores Comunes

<Aside type="danger" title="Escribir fourty en vez de forty">
- **Fourty**, fourty-five ⛔
- **Forty**, forty-five ✅

El número 40 es la única decena que pierde la "u" del número base "four".
</Aside>

<Aside type="danger" title="Pluralizar hundred, thousand o million con números específicos">
- Two **hundreds**, five **thousands**, ten **millions** ⛔
- Two **hundred**, five **thousand**, ten **million** ✅

Cuando van precedidos de un número específico, NO se pluralizan. Solo se pluralizan cuando se usan de forma indefinida: "hundreds of people", "thousands of dollars".
</Aside>

<Aside type="danger" title="Confundir teen con ty por el acento">
- Confundir "fifteen" (15) con "fifty" (50) debido al acento ⛔
- **Thirteen** /θɜːrˈtiːn/ → acento en "TEEN" (13) ✅
- **Thirty** /ˈθɜːrti/ → acento en "THIR" (30) ✅

La diferencia está en el acento: "-teen" tiene acento al final, "-ty" al principio. Exagera la pronunciación de la sílaba acentuada para evitar confusiones.
</Aside>
